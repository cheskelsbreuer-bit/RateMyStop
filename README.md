## 1. Run it locally on your machine

### Step 1 — Install Python (one-time)

You don't have Python yet. Install it from https://www.python.org/downloads/windows/ — pick the
latest 3.12.x installer. **Check the box "Add python.exe to PATH"** during install.

After install, open a **new** terminal and confirm:
```bat
python --version
```
You should see `Python 3.12.x`.

### Step 2 — Start the backend

Double-click `backend\run_local.bat` (or run it from a terminal). It will:
1. Create a Python virtual environment in `backend/.venv`
2. Install FastAPI + dependencies
3. Copy `.env.example` to `.env`
4. Seed the demo officer data
5. Start the API server on http://localhost:8000

API docs (auto-generated by FastAPI): http://localhost:8000/docs

### Step 3 — Start the frontend

In a **second terminal**, double-click `frontend\run_local.bat`. It serves the frontend on
http://localhost:5500.

Open http://localhost:5500 in your browser. The site is now talking to your Python backend.

## 2. Deploy to the real internet

### Backend → Railway (free tier)

1. Push this repo to GitHub.
2. Go to https://railway.app → **New Project → Deploy from GitHub**.
3. Pick your repo, then set **Root Directory** to `backend`.
4. Railway auto-detects Python and reads `railway.toml` for the start command.
5. Add a **PostgreSQL** plugin (one click). Railway sets `DATABASE_URL` automatically.
6. Add these env vars in the Railway dashboard:
   - `CORS_ORIGINS=https://YOUR-FRONTEND.vercel.app`
   - `RESEND_API_KEY=...` (optional — see step 3)
   - `RESEND_FROM=RateMyStop <complaints@yourdomain.com>` (optional)
7. Hit **Deploy**. You'll get a URL like `https://ratemystop-backend.up.railway.app`.

> ⚠️ Railway's filesystem is ephemeral — uploaded ticket photos will disappear on redeploy.
> For production, swap `/api/uploads` to write to **Cloudflare R2** or **AWS S3**. I can wire
> that in when you're ready.

### Frontend → Vercel (free)

1. Go to https://vercel.com → **New Project** → import your GitHub repo.
2. Set **Root Directory** to `frontend`.
3. Framework preset: **Other** (it's plain static HTML).
4. Before deploying, add this snippet to the top of `frontend/index.html` (right after the
   opening `<head>`) so it knows where the backend lives:
   ```html
   <script>window.API_BASE = 'https://YOUR-BACKEND.up.railway.app';</script>
   ```
5. Deploy. You'll get `https://ratemystop.vercel.app`.

### Custom domain (optional)

Buy `ratemystop.com` (Namecheap / Cloudflare Registrar / etc.). In Vercel → Project →
Settings → Domains, add it and follow the DNS instructions.

## 3. Email sending (complaints) — Resend

Without this, complaints are still saved to the DB but no emails go out.

1. Sign up at https://resend.com (free: 3,000 emails/month).
2. Verify a domain you control (or use their test sender for development).
3. Set `RESEND_API_KEY` and `RESEND_FROM` env vars on Railway.

## 4. API reference

| Method | Path                       | What it does |
|--------|----------------------------|----|
| GET    | `/api/officers`            | List all officers with their summary stats |
| GET    | `/api/officers/{id}`       | One officer's full detail + reviews |
| POST   | `/api/reviews`             | Submit a review (creates officer if new) |
| POST   | `/api/uploads`             | Upload a ticket photo, returns `{url}` |
| POST   | `/api/complaints`          | File a complaint (saves to DB + emails) |
| GET    | `/api/stats`               | Homepage numbers (totals, % unfair, avg ticket) |
| GET    | `/health`                  | Health check |
| GET    | `/docs`                    | Swagger UI (auto-generated) |

## 5. Important legal notes

This site publishes named officers with crowd-sourced "fair / unfair" verdicts. Before
launching publicly:

- Add a **moderation queue** for new reviews (don't auto-publish).
- Add an officer **dispute / takedown form**.
- Consider requiring some form of verification (uploaded ticket photo) before publishing.
- Talk to a lawyer about defamation exposure under your state's laws.

The current code accepts and shows reviews immediately — fine for staging / closed beta,
risky for public launch.
